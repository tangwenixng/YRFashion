# 小程序端功能清单

版本：v0.1  
日期：2026-03-14

## 1. 目标说明

本文基于 [04-MVP架构设计-Python后端版.md](D:\Codes\YRFashion\docs\04-MVP架构设计-Python后端版.md) 中与“小程序端”相关的设计内容，整理第一版小程序的功能清单。

本清单只覆盖第一版 MVP 所需的用户侧能力，不包含后台管理端功能。

## 2. 小程序端目标

第一版小程序只聚焦三件事：

- 商品展示
- 商品详情查看
- 商品留言/咨询

第一版不做：

- 支付
- 购物车
- 订单
- 优惠券
- 复杂会员体系
- 收藏/预约
- 订阅通知

## 3. 页面清单

建议第一版小程序至少包含以下页面：

### 3.1 首页【已完成】

功能项：

- 展示店铺基础信息
- 展示首页 banner
- 展示主推商品入口
- 展示店铺简介摘要
- 展示联系方式入口

数据来源：

- `GET /api/miniapp/home`
- 店铺设置中的 `shop_name`、`shop_intro`、`homepage_banner_json`

### 3.2 商品列表页【已完成】

功能项：

- 按已上架状态展示商品
- 按 `sort_order` 排序展示
- 展示商品封面图
- 展示商品名称
- 展示商品标签
- 支持进入商品详情页

建议增强：

- 支持分页加载
- 支持下拉刷新

数据来源：

- `GET /api/miniapp/products`

### 3.3 商品详情页【已完成】

功能项：

- 展示商品名称
- 展示商品图片轮播
- 展示商品描述
- 展示商品标签
- 展示留言/咨询入口
- 展示联系方式快捷入口

数据来源：

- `GET /api/miniapp/products/{id}`

### 3.4 留言提交页或详情页内留言区【已完成】

功能项：

- 输入留言内容
- 提交对当前商品的咨询
- 提交成功后给出反馈
- 控制留言内容长度
- 为后续敏感词过滤预留提示

数据来源：

- `POST /api/miniapp/products/{id}/messages`

说明：

- 一期是“用户留言，店主后台回复”
- 小程序端第一版不强制展示回复记录，可先只完成留言提交闭环

### 3.5 联系方式页【已完成】

功能项：

- 展示联系电话
- 展示微信号
- 展示门店地址
- 展示营业时间
- 支持拨打电话
- 支持复制微信号

数据来源：

- `GET /api/miniapp/shop/contact`

## 4. 通用能力清单

### 4.1 小程序登录与登录态【已完成】

功能项：

- 调用 `wx.login()` 获取 `code`
- 将 `code` 发送给后端换取本站 token
- 本地保存 access token
- 后续请求自动带 token
- 首次访问自动建档
- 更新最近访问时间

接口依赖：

- `POST /api/miniapp/auth/login`

说明：

- 以 `openid` 作为用户唯一外部标识
- 不把昵称、头像作为强依赖

### 4.2 页面导航【已完成】

功能项：

- 首页进入商品列表
- 商品列表进入商品详情
- 商品详情进入留言
- 首页进入联系方式

建议：

- 第一版导航保持简单，不做复杂多 tab 体系

### 4.3 数据加载与异常处理【已完成】

功能项：

- 页面首次进入自动加载
- 请求失败时给出 toast 或空态提示
- 无数据时展示空态
- 网络慢时展示 loading

### 4.4 分享能力【已完成】

功能项：

- 商品详情支持转发
- 分享卡片默认展示商品名称和封面图

说明：

- 这属于微信小程序原生能力，建议第一版直接接入

## 8. 非功能要求

### 8.1 性能【已完成（基础实现）】

- 首页和商品列表加载尽量轻量
- 商品图片优先使用压缩图
- 列表接口建议分页

### 8.2 安全【已完成（基础实现）】

- 所有业务请求走 HTTPS
- 小程序只保存业务 token，不暴露微信敏感信息
- 留言内容做长度限制

### 8.3 可维护性【已完成（基础实现）】

- 页面结构保持简单
- 数据字段尽量与后端接口一一对应
- 为后续收藏、预约、订阅通知预留扩展空间

## 10. 一句话结论

第一版小程序端的核心不是“交易”，而是“展示 + 咨询 + 联系”。只要把登录、商品列表、商品详情、留言和联系方式五条链路打通，就满足当前 MVP 目标。
