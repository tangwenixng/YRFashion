很多小程序**内容和布局随时变化**，背后通常不是把页面写死在前端，而是通过**后端或配置系统控制**。常见有三种实现方式，从简单到高级我给你讲一下。

---

# 1️⃣ 最常见：后端返回数据控制内容

这是**绝大多数小程序**用的方式。

小程序只负责渲染，真正的内容来自服务器 API。

例如小程序请求：

```http
GET https://api.tangwx.site/home
```

服务器返回：

```json
{
  "banner":[
    {"img":"a.jpg","link":"/pages/a"}
  ],
  "articles":[
    {"title":"新闻1"},
    {"title":"新闻2"}
  ]
}
```

小程序代码只是：

```javascript
wx.request({
  url: "https://api.tangwx.site/home"
})
```

优点：

* 内容随时能变
* 不需要重新发布小程序
* 后台改数据即可

---

# 2️⃣ 配置化页面（布局也能变）

更高级一点的做法是：
**后端返回页面结构**。

例如返回：

```json
{
  "components":[
    {"type":"banner"},
    {"type":"grid"},
    {"type":"article_list"}
  ]
}
```

小程序根据 `type` 渲染组件：

```javascript
switch(type){
  case "banner":
  case "grid":
  case "article_list":
}
```

这样后台可以控制：

* 页面顺序
* 是否显示
* 内容

很多电商首页就是这么做的。

# 3️⃣ CMS / 运营后台（大公司常用）

运营人员可以：

* 拖拽组件

这种模式在：

特别常见。

## 1️⃣ 布局写死在前端代码里

比如你的小程序页面是这样写的：

```xml
<view class="banner"></view>
<view class="menu"></view>
<view class="article"></view>
```

如果你想改成：

```xml

<view class="banner"></view>
<view class="article"></view>
```

或者新增模块：

```xml
<view class="banner"></view>
<view class="menu"></view>
<view class="activity"></view>
<view class="article"></view>
```

那就必须：

1. 修改代码
2. 上传新版本
3. 在 微信公众平台 提交审核
4. 审核通过后发布

一般审核时间：

* 快：1~2小时
* 常见：几个小时
* 偶尔：1天

所以如果**布局写死**，每次结构变化确实要重新审核。

---

## 2️⃣ 布局固定，但内容来自后端（最常见）

很多小程序其实是这样设计的：

前端布局：


后端返回：

```json

  "articles":[...]
}
```

这种情况下你可以随时：

* 换 banner
* 改文章
* 改商品
* 改图片

都 **不需要重新审核**。

## 3️⃣ 高级做法：布局也由后端控制

* 调整模块顺序

**也不需要重新审核**。

# 更好的架构（推荐一开始就这么做）

把首页做成 **模块化渲染**。

服务器返回：

```json
{
  "modules":[
    {"type":"banner"},
    {"type":"menu"},
    {"type":"article"},
    {"type":"video"}
  ]
}
```

小程序循环渲染：

```xml
<block wx:for="{{modules}}">
  <component is="{{item.type}}" />
</block>
```

这样：

服务器控制：

```json
{
  "modules":[
    {"type":"banner"},
    {"type":"activity"},
    {"type":"menu"},
    {"type":"article"}
  ]
}
```

页面顺序就变了。

**小程序不用改代码。**

* 新增模块
* 隐藏模块

全部通过后台完成。

不用：

* 提交审核
* 改前端代码
