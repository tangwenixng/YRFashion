# 小程序管理端一期实施清单

版本：v0.1  
日期：2026-03-15

## 1. 目标说明

本文用于定义“小程序管理端”第一期的建设范围。

总体原则：

- 保留现有 `frontend` 作为完整 Web 管理后台
- 在 `miniapp` 中补充一套轻量移动管理端
- 小程序管理端只承载高频、现场、移动端更方便的操作
- 复杂报表、低频配置、重表格操作继续保留在 Web 端

## 2. 一期目标

第一期只解决两个核心问题：

- 管理人员可直接用手机登录后台账号
- 管理人员可直接用手机完成商品维护和留言处理

第一期不追求替代 Web 后台，只追求形成移动端最小闭环。

## 3. 一期范围

### 3.1 管理员登录

- 使用现有后台账号密码登录
- 与普通小程序用户登录态分离
- 独立保存管理员 token
- 管理页面进入时校验管理员登录态

### 3.2 商品管理

- 商品列表查看
- 商品新增
- 商品编辑
- 商品上下架
- 商品图片上传
- 商品图片删除
- 商品图片设为封面
- 商品图片排序调整

### 3.3 留言管理

- 留言列表查看
- 按状态筛选留言
- 查看留言详情
- 标记已读 / 未读
- 回复留言

## 4. 暂不纳入一期

- 仪表盘报表
- 用户列表与用户分析
- 通知配置
- 店铺全量设置
- 复杂批量操作
- 适合大屏操作的长表单和复杂表格

## 5. 小程序页面清单

建议在 `miniapp` 中新增管理员分包，页面如下：

- `pages-admin/login/index`
- `pages-admin/home/index`
- `pages-admin/products/index`
- `pages-admin/product-form/index`
- `pages-admin/product-images/index`
- `pages-admin/messages/index`
- `pages-admin/message-detail/index`

## 6. 后端接口复用清单

第一期优先复用现有后台接口，不新增独立的“miniapp-admin”后端接口。

### 6.1 认证

- `POST /api/admin/auth/login`
- `GET /api/admin/auth/me`

### 6.2 商品

- `GET /api/admin/products`
- `POST /api/admin/products`
- `GET /api/admin/products/{id}`
- `PUT /api/admin/products/{id}`
- `PUT /api/admin/products/{id}/sort`
- `POST /api/admin/products/{id}/images`
- `POST /api/admin/products/{productId}/images/{imageId}/cover`
- `PUT /api/admin/products/{productId}/images/sort`
- `DELETE /api/admin/products/{productId}/images/{imageId}`

### 6.3 分类

- `GET /api/admin/categories`

### 6.4 留言

- `GET /api/admin/messages`
- `POST /api/admin/messages/{id}/read`
- `POST /api/admin/messages/{id}/unread`
- `POST /api/admin/messages/{id}/reply`

## 7. 小程序端实现建议

### 7.1 目录结构

- 管理员页面放入分包，避免与普通用户页面混杂
- 新增管理员独立请求层和鉴权层
- 普通用户态与管理员态不要共用 token 存储

建议新增：

- `miniapp/utils/admin-auth.js`
- `miniapp/utils/admin-http.js`
- `miniapp/utils/admin-api/*.js`

### 7.2 登录态设计

- 普通用户继续走现有 `miniapp/auth/login`
- 管理员继续走现有 `admin/auth/login`
- 两套 token 分别存储
- 管理页面只认管理员 token

### 7.3 权限边界

- 任何管理员页面都应在进入时校验管理员身份
- 普通用户页面不依赖管理员登录态
- 即使用户看得到“管理员入口”，没有管理员账号也无法进入实际管理能力

## 8. 一期实施顺序

建议按以下顺序编码：

1. 管理员登录与路由入口
2. 商品列表
3. 商品新增 / 编辑
4. 图片上传与封面 / 排序 / 删除
5. 留言列表
6. 留言详情与回复
7. 真机回归测试

## 9. 验收标准

一期完成后，应满足：

- 管理员可在手机上登录并保持登录态
- 可新增商品并编辑商品信息
- 可上传图片并设置封面
- 可调整商品上下架状态
- 可查看留言并回复
- 普通用户端现有功能不受影响
- Web 后台继续可用

## 10. 一句话结论

小程序管理端第一期不是“把 Web 后台搬进手机”，而是补齐“商品管理 + 留言处理”这条最有价值的移动端闭环。
