# 小程序本地联调说明
版本：v0.2  
日期：2026-03-14

## 1. 适用范围

本文用于当前仓库内以下两个工程的联调与测试：

- `backend`：FastAPI 后端
- `miniapp`：原生微信小程序

当前仓库路径：

- `backend`
- `miniapp`

## 2. 本地环境准备

建议先确认本机已安装：

- Python 3.12
- `uv`
- Node.js
- 微信开发者工具

如果 `backend/.venv` 尚未创建，先执行：

```powershell
cd .\backend
uv sync
```

## 3. 启动后端

在项目根目录执行：

```powershell
cd .\backend
uv run uvicorn backend.main:app --reload
```

默认启动后：

- 服务地址：`http://127.0.0.1:8000`
- API 前缀：`http://127.0.0.1:8000/api`
- Swagger 文档：`http://127.0.0.1:8000/docs`

默认数据位置：

- SQLite：`backend/data/app.db`
- 上传目录：`backend/data/uploads`
- 管理后台默认账号：`admin`
- 管理后台默认密码：`admin123456`

## 4. 打开小程序工程

使用微信开发者工具导入目录：

- [miniapp](/D:/Codes/YRFasion/miniapp)

当前小程序接口配置文件：

- [config.js](/D:/Codes/YRFasion/miniapp/utils/config.js)

默认配置：

```js
const API_BASE_URL = "http://127.0.0.1:8000/api"
```

如果本地后端端口变化，直接修改该文件即可。

## 5. 开发者工具配置

本地联调默认走 `http://127.0.0.1:8000`，通常需要在微信开发者工具中关闭域名校验。

建议检查：

1. 打开“详情”
2. 勾选“不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书”
3. 重新编译小程序

如果不关闭校验，本地 `HTTP` 接口通常会被拦截。

## 8. 登录说明

当前小程序登录仍是开发期 mock 方案，不是真实微信 `code2Session`。

相关实现：

- [auth.js](/D:/Codes/YRFasion/miniapp/utils/auth.js)
- [miniapp_auth.py](/D:/Codes/YRFasion/backend/src/backend/api/routes/miniapp_auth.py)
- [miniapp_auth.py](/D:/Codes/YRFasion/backend/src/backend/services/miniapp_auth.py)

当前行为：

1. 小程序调用 `wx.login()`
2. 后端接收 `code`
3. 后端根据 `code` 生成稳定 mock `openid`
4. 返回业务 JWT token

该方案适合本地开发与功能联调，不适合正式环境。后续正式接入时，需要把 `resolve_openid_from_code()` 替换成真实微信 `code2Session` 调用。

## 10. 真机调试前需要准备什么

真机调试的核心问题不是代码本身，而是“手机能不能访问后端”。

当前小程序接口配置为：

```js
const API_BASE_URL = "http://127.0.0.1:8000/api"
```

这在开发者工具模拟器中可用于本地联调，但在真机上不可直接使用。原因是：

- `127.0.0.1` 在真机上指向手机自己
- 手机不会访问到你电脑上的后端服务

真机调试前建议准备：

- 一个可用的 `AppID`
- 当前微信号具备该小程序开发者或体验成员权限
- 微信开发者工具
- 一台手机可访问到的后端服务
- 如果走局域网方案，确保手机与电脑在同一 Wi-Fi
- 如果走公网方案，准备好 `HTTPS` 域名

## 11. 真机调试的两种推荐方案

### 11.1 局域网真机联调

适合当前开发阶段，配置最快。

步骤：

1. 使用 `0.0.0.0` 启动后端

```powershell
cd .\backend
uv run uvicorn backend.main:app --host 0.0.0.0 --port 8000 --reload
```

2. 查询电脑局域网 IP，例如 `192.168.1.23`
3. 修改 [config.js](/D:/Codes/YRFasion/miniapp/utils/config.js)

```js
const API_BASE_URL = "http://192.168.1.23:8000/api"
```

4. 保证手机和电脑在同一 Wi-Fi
5. 放行电脑防火墙中的 `8000` 端口
6. 在微信开发者工具中重新编译，再执行“真机调试”或“预览”

说明：

- 该方案适合日常开发联调
- 不适合作为正式上线前环境

### 11.2 HTTPS 域名真机联调

适合接近真实环境的测试。

步骤：

1. 为后端准备一个公网可访问的 `HTTPS` 地址
2. 修改 [config.js](/D:/Codes/YRFasion/miniapp/utils/config.js)

```js
const API_BASE_URL = "https://你的域名/api"
```

3. 登录微信小程序后台
4. 配置服务器域名
5. 重新编译并进行真机调试

该方案的优点：

- 更接近正式环境
- 更稳定
- 后续提审与上线迁移成本更低

## 12. 微信后台通常需要处理的流程

如果走真机调试或后续提审，通常需要在微信后台处理：

1. 登录小程序后台
2. 进入开发相关设置
3. 配置服务器域名

通常需要关注：

- `request` 域名
- 如果后续有上传能力，还需要 `uploadFile`
- 如果后续有下载能力，还需要 `downloadFile`

当前项目中，图片资源也依赖接口基址派生的媒体地址，所以接口域名切换后，商品图片地址也会随之生效。

## 13. 常见问题

### 13.1 小程序请求失败

优先检查：

- 后端是否已启动
- [config.js](/D:/Codes/YRFasion/miniapp/utils/config.js) 中端口或域名是否正确
- 微信开发者工具是否关闭了合法域名校验

### 13.2 首页没有商品

优先检查：

- 商品是否为 `published`
- 商品是否有封面图
- 是否已准备店铺信息和已发布商品

### 13.3 图片不显示

优先检查：

- 上传文件是否实际存在于 `backend/data/uploads`
- 数据库中 `image_url` 是否正常
- 小程序是否已重新编译
- 当前接口域名是否能被手机访问
- 小程序图片地址归一化逻辑是否正常

### 13.4 留言提交失败

优先检查：

- 当前商品是否为已发布状态
- 小程序是否已成功登录
- 留言内容是否为空或超长

### 13.5 真机上图片或接口访问失败

优先检查：

- 是否还在使用 `127.0.0.1`
- 手机和电脑是否在同一网络
- 后端是否以 `0.0.0.0` 启动
- 电脑防火墙是否阻止了 `8000` 端口
- 如果使用 `HTTPS` 域名，微信后台是否已配置合法域名

## 14. 推荐联调顺序

建议按以下顺序排查：

1. 启动后端并打开 `/docs`
2. 确认已有店铺配置和已发布商品
3. 导入 [miniapp](/D:/Codes/YRFasion/miniapp)
4. 检查 [config.js](/D:/Codes/YRFasion/miniapp/utils/config.js) 中接口地址
5. 开发者工具中关闭域名校验
6. 先验证首页和商品列表
7. 再验证商品详情、留言提交和留言记录展示
8. 如需真机调试，再切换为局域网 IP 或 `HTTPS` 域名
