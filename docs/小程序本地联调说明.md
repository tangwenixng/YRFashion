# 小程序本地联调说明

版本：v0.1  
日期：2026-03-14

## 1. 适用范围

本文用于本地联调当前仓库中的：

- FastAPI 后端
- 原生微信小程序端

当前仓库路径：

- `backend`：Python 后端
- `miniapp`：微信小程序工程

## 2. 本地环境准备

建议先确认本机已安装：

- Python 3.12
- `uv`
- Node.js
- 微信开发者工具

如果 `backend/.venv` 尚未创建，先在项目根目录执行：

```powershell
cd .\backend
uv sync
```

## 3. 启动后端

在项目根目录执行：

```powershell
cd .\backend
uv run uvicorn backend.main:app --reload
```

默认启动后：

- 服务地址：`http://127.0.0.1:8000`
- API 前缀：`http://127.0.0.1:8000/api`
- Swagger 文档：`http://127.0.0.1:8000/docs`

本地默认数据：

- SQLite：`backend/data/app.db`
- 上传目录：`backend/data/uploads`
- 管理后台默认账号：`admin`
- 管理后台默认密码：`admin123456`

## 4. 打开小程序工程

使用微信开发者工具导入目录：

- [miniapp](/D:/Codes/YRFasion/miniapp)

当前小程序请求地址配置在：

- [miniapp/utils/config.js](/D:/Codes/YRFasion/miniapp/utils/config.js)

默认值为：

```js
const API_BASE_URL = "http://127.0.0.1:8000/api"
```

如果你本地后端端口变了，直接修改这个文件即可。

## 5. 开发者工具配置

因为当前本地联调使用的是 `http://127.0.0.1:8000`，通常需要在微信开发者工具中关闭域名校验。

建议在开发者工具中检查以下设置：

1. 打开“详情”。
2. 勾选“不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书”。
3. 重新编译小程序。

如果不关闭校验，小程序请求本地 HTTP 接口时会被拦截。

## 6. 当前联调链路

现在已经可以在本地打通以下页面和接口：

- 小程序登录
- 首页
- 商品列表页
- 商品详情页
- 留言提交页
- 联系方式页

对应接口：

- `POST /api/miniapp/auth/login`
- `GET /api/miniapp/home`
- `GET /api/miniapp/products`
- `GET /api/miniapp/products/{id}`
- `POST /api/miniapp/products/{id}/messages`
- `GET /api/miniapp/shop/contact`

## 7. 联调前建议准备的数据

小程序首页和商品页依赖后台已有数据。建议先准备：

1. 至少 1 条店铺设置
2. 至少 2 个已发布商品
3. 至少给其中 1 个商品上传封面图

你可以直接通过接口文档或现有管理后台维护这些数据。

如果商品没有发布：

- 商品列表页不会显示
- 首页主推商品区域也可能为空

## 8. 登录说明

当前小程序登录使用的是开发态 mock 方案，不是正式微信 `code2Session`。

实现位置：

- [miniapp/utils/auth.js](/D:/Codes/YRFasion/miniapp/utils/auth.js)
- [backend/src/backend/api/routes/miniapp_auth.py](/D:/Codes/YRFasion/backend/src/backend/api/routes/miniapp_auth.py)
- [backend/src/backend/services/miniapp_auth.py](/D:/Codes/YRFasion/backend/src/backend/services/miniapp_auth.py)

当前行为是：

- 小程序调用 `wx.login()`
- 后端接收 `code`
- 后端根据 `code` 生成一个稳定的 mock `openid`
- 返回业务 JWT token

这适合本地开发和功能联调，但不适合正式环境。

后续正式接入时，需要把 `resolve_openid_from_code()` 替换成真实微信 `code2Session` 调用。

## 10. 常见问题

### 10.1 小程序请求失败

优先检查：

- 后端是否已经启动
- `miniapp/utils/config.js` 中端口是否正确
- 微信开发者工具是否关闭了合法域名校验

### 10.2 首页没有商品

优先检查：

- 商品是否为 `published`
- 商品是否已设置排序
- 是否至少有一条商品数据

### 10.3 图片不显示

优先检查：

- 上传文件是否实际存在于 `backend/data/uploads`
- 数据库中的 `image_url` 是否以 `/uploads/...` 返回
- 后端静态目录是否正常挂载

### 10.4 留言提交失败

优先检查：

- 当前商品是否为已发布状态
- 小程序是否已成功登录
- 留言内容是否为空或超长

## 11. 推荐联调顺序

建议按下面顺序排查：

1. 启动后端并打开 `/docs`
2. 确认后台已有店铺设置和已发布商品
3. 微信开发者工具导入 `miniapp`
4. 关闭合法域名校验
5. 先验证首页和商品列表
6. 再验证商品详情和留言提交
